This endpoint returns every agency record stored in the database, regardless of activation state. It is the primary way to inspect which carriers are configured, review their capability rules, and confirm that `apiConfig` and `supplements` are set correctly before running a rate comparison. Use the `active` query parameter to narrow results when you only need carriers currently participating in quotes.

<Info>
  The rate comparison engine (`POST /api/v1/rates/compareByPostalCode`) automatically considers only active agencies. This listing endpoint intentionally returns all agencies so you can audit inactive ones without a separate lookup.
</Info>

## Endpoint

```
GET /api/v1/agencies
```

## Query Parameters

<ParamField query="active" type="boolean">
  Filter agencies by their activation state.

  | Value | Effect |
  |-------|--------|
  | `true` | Returns only agencies where `active` is `true`. |
  | `false` | Returns only agencies where `active` is `false`. |
  | *(omitted)* | Returns all agencies regardless of state. |
</ParamField>

## Response

A successful request returns HTTP **200 OK** with a JSON array of agency objects. Each item in the array has the same shape as the object returned by `POST /api/v1/agencies`.

<ResponseField name="[]" type="array">
  Array of agency objects.

  <Expandable title="Agency object fields">
    <ResponseField name="id" type="string">
      MongoDB ObjectId of the agency.
    </ResponseField>

    <ResponseField name="name" type="string">
      Title-cased human-readable agency name (e.g. `"Dachser"`, `"Mrw"`).
    </ResponseField>

    <ResponseField name="code" type="string">
      Normalised slug derived from `name` (e.g. `"dachser"`, `"mrw"`). Unique across all agencies.
    </ResponseField>

    <ResponseField name="type" type="string">
      Rate sourcing strategy: `"static"`, `"api"`, or `"hybrid"`.
    </ResponseField>

    <ResponseField name="active" type="boolean">
      `true` if the agency is included in rate comparison results.
    </ResponseField>

    <ResponseField name="rules" type="object">
      Routing and capability rules for the agency.

      <Expandable title="rules fields">
        <ResponseField name="hasAndaluciaRule" type="boolean">
          Whether special Andalusia pricing is applied.
        </ResponseField>
        <ResponseField name="supportsPallets" type="boolean">
          Whether the agency handles pallet shipments.
        </ResponseField>
        <ResponseField name="supportsParcels" type="boolean">
          Whether the agency handles parcel shipments.
        </ResponseField>
        <ResponseField name="coverage" type="array of strings">
          Geographic scope: `["national"]`, `["international"]`, or `["national", "international"]`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="supplements" type="object">
      Pricing supplements applied on top of the base rate.

      <Expandable title="supplements.fuelSurcharge fields">
        <ResponseField name="fuelSurcharge.enabled" type="boolean">
          Whether the fuel surcharge is active.
        </ResponseField>
        <ResponseField name="fuelSurcharge.type" type="string">
          `"percentage"` or `"fixed"`.
        </ResponseField>
        <ResponseField name="fuelSurcharge.value" type="number">
          Surcharge amount (percentage points or fixed euro value).
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="apiConfig" type="object">
      Carrier API connection details. Present on `"api"` and `"hybrid"` agencies.

      <Expandable title="apiConfig fields">
        <ResponseField name="baseUrlApi" type="string">
          Base URL of the carrier's external API.
        </ResponseField>
        <ResponseField name="timeout" type="number">
          Request timeout in milliseconds.
        </ResponseField>
        <ResponseField name="apiKey" type="string">
          Authentication key for the carrier API.
        </ResponseField>
        <ResponseField name="endpoints" type="object">
          <Expandable title="endpoints fields">
            <ResponseField name="quotations" type="string">
              Path for quotation requests.
            </ResponseField>
            <ResponseField name="transportOrders" type="string">
              Path for transport order creation.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="createdAt" type="string">
      ISO 8601 creation timestamp.
    </ResponseField>

    <ResponseField name="updatedAt" type="string">
      ISO 8601 last-update timestamp.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<ResponseField name="404 Not Found" type="object">
  Returned when no agency documents exist in the database.

  ```json
  {
    "message": "Agencies not found"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error" type="object">
  Returned on unexpected database or server errors.

  ```json
  {
    "message": "Error internal server"
  }
  ```
</ResponseField>
